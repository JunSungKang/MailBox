# 장애대응가이드

## 이미지 pull 실패

- registry 주소와 이미지 태그를 `mail-server.env`에서 확인한다.
- private registry라면 `docker login` 상태를 확인한다.
- 폐쇄망이면 이미지 tar 파일을 `docker load`로 먼저 적재한다.

## 컨테이너 기동 실패

```bash
mail-server/compose.sh ps
mail-server/compose.sh logs mail-smtp-receiver
mail-server/compose.sh logs mail-imap-mailbox
mail-server/compose.sh logs mail-webmail-ui
```

포트 충돌이 있으면 `SMTP_PORT`, `IMAP_PORT`, `MAIL_WEB_PORT` 값을 변경한다.

## 메일 수신 실패

- DNS `MX` 레코드가 서버를 가리키는지 확인한다.
- 클라우드 방화벽과 OS 방화벽에서 TCP 25번 인바운드가 열려 있는지 확인한다.
- `mail-smtp-receiver` 로그에서 수신 주소 매핑 실패 여부를 확인한다.

## 웹메일 접속 실패

- `mail-webmail-ui` 컨테이너 상태와 로그를 확인한다.
- `MAIL_WEB_PORT`가 방화벽에서 허용되어 있는지 확인한다.
- `checks/healthcheck.sh`로 HTTP 응답을 확인한다.
