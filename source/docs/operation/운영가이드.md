# 운영가이드

## 상태 확인

```bash
mail-server/compose.sh ps
mail-server/compose.sh logs -f mail-smtp-receiver
mail-server/compose.sh logs -f mail-imap-mailbox
mail-server/compose.sh logs -f mail-webmail-ui
```

## 로그 확인

Compose 실행 시 SMTP 컨테이너의 `/var/log`를 호스트 로그 디렉터리에 마운트한다. 기본 경로는 패키지의 `mail-server/logs/smtp`이며, `mail-server.env`의 `MAIL_LOG_DIR` 값으로 변경할 수 있다.

```bash
tail -f mail-server/logs/smtp/postfix.log
```

IMAP과 Webmail은 표준 출력 로그를 사용한다. Compose 로그는 local logging driver로 보존하며, `MAIL_LOG_MAX_SIZE`, `MAIL_LOG_MAX_FILE` 값으로 회전 크기와 파일 개수를 조정한다.

```bash
mail-server/compose.sh logs -f mail-imap-mailbox
mail-server/compose.sh logs -f mail-webmail-ui
```

## 재시작

```bash
mail-server/compose.sh restart
```

## 버전 교체

1. `mail-server.env`의 이미지 태그를 새 릴리즈 값으로 변경한다.
2. `images.lock`의 이미지 목록과 비교한다.
3. 이미지를 pull하고 컨테이너를 재생성한다.

```bash
mail-server/compose.sh pull
mail-server/compose.sh up -d
checks/healthcheck.sh
```

## 롤백

이전 릴리즈 패키지의 `mail-server.env` 이미지 태그로 되돌린 뒤 같은 절차로 `pull`, `up -d`, `healthcheck`를 실행한다.

## 이미지 빌드 및 Push

이미지를 새로 배포해야 하는 작업은 소스 저장소에서 수행한다. 납품 패키지에는 소스 코드와 Dockerfile이 포함되지 않는다.

전체 이미지를 같은 버전 태그로 다시 빌드하고 GHCR에 push하려면 다음 순서로 진행한다.

```bash
cd source
cp scripts/images/images.env.example scripts/images/images.env
vi scripts/images/images.env
```

`scripts/images/images.env`에는 배포할 registry, namespace, version, GHCR 계정 정보를 설정한다.

```dotenv
REGISTRY=ghcr.io
IMAGE_NAMESPACE=junsungkang
VERSION=<release_tag>
IMAGE_NAME_PREFIX=mail-server-

GHCR_USERNAME=JunSungKang
GHCR_TOKEN=<github_package_write_token>
```

토큰에는 GHCR package write 권한이 필요하다. `scripts/images/images.env`는 커밋하지 않는다.

```bash
scripts/images/build-images.sh
scripts/images/push-images.sh
scripts/images/write-images-lock.sh
```

특정 이미지만 삭제되었거나 태그만 복구해야 하면 같은 태그로 해당 이미지만 빌드하고 push할 수 있다.

```bash
docker build -t ghcr.io/junsungkang/mail-server-webmail-ui:<release_tag> apps/webmail-ui
docker push ghcr.io/junsungkang/mail-server-webmail-ui:<release_tag>
docker pull ghcr.io/junsungkang/mail-server-webmail-ui:<release_tag>
```

복구 후 운영 서버에서는 `mail-server.env`의 이미지 태그가 복구한 태그와 같은지 확인하고 `mail-server/compose.sh pull`을 다시 실행한다.

## 수신 주소 관리

수신 주소 파일은 이미지 빌드 시 포함된다. 운영 중 주소 변경이 필요하면 소스 저장소의 `source/apps/smtp-receiver/virtual-mailbox-accounts`를 수정하고 새 이미지를 빌드해 배포한다.
